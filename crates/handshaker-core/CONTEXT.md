# gRPC Core

OS-независимое ядро Handshaker: контракт метода (reflection), вызов (invoke),
библиотека коллекций, окружения, auth и резолв `{{var}}`-шаблонов. Ядро не знает
про Tauri/specta и про UI.

## Language

### Резолв запроса

**Резолв-пайплайн (resolve pipeline)**:
Единственное преобразование «сохранённый запрос или черновик + коллекция +
активное окружение → effective request». Живёт в ядре; никакая другая поверхность
не повторяет его шаги.
_Avoid_: «фронтовый резолв», sendStep-резолв (исторический TS-двойник, удалён)

**Effective request**:
Полностью разрешённый, готовый к отправке запрос: target (адрес+TLS), тело,
metadata, материализованный auth-заголовок. Результат резолв-пайплайна.
_Avoid_: resolved request, InvokeRequest (историческая проводная форма, удалена)

**Resolve failure**:
Типизированный отказ резолва: полный (дедуплицированный, в порядке встречи)
список неразрешённых переменных плюс цепочка цикла, если он есть. Не «первая
ошибка», а весь диагноз разом.
_Avoid_: UnresolvedVariable (одиночная ошибка старого строгого резолва)

**Dynamic var (билтин)**:
`{{$guid}}`-подобная переменная, генерируемая в момент Send (per-occurrence).
Резолв-пайплайн её не считает неразрешённой — она остаётся литералом до фазы
экспансии билтинов. Пользовательская переменная того же имени побеждает.
Экспансия применяется только к пользовательским шаблонным полям (body, значения
metadata); материализованный auth-заголовок — факт, не шаблон, и не экспандится.
_Avoid_: unresolved var (это другой bucket)

### Коллекции

**Collection link (ссылка коллекции)**:
Именованный внешний URL, принадлежащий коллекции, — быстрый переход к
инструментам сервиса, который коллекция представляет (Grafana, логи, дока).
URL — `{{var}}`-шаблон, резолвится против активного окружения; с неразрешёнными
переменными переход заблокирован (ссылка помечена, показан отсутствующий ключ).
Носитель — только коллекция; папки и запросы ссылок не имеют.
_Avoid_: service explorer (рабочее название фичи, не термин модели);
service link (носитель — Collection, а не gRPC-сервис из reflection)

### Auth

**Auth pick (выбор auth)**:
Чистый синхронный выбор выигрывающего auth-конфига: ближайший активный
не-`None` по цепочке request → collection, с env-гейтом (конфиг, чей список
окружений не содержит активное, пропускается как отсутствующий). Правило живёт
в ядре ровно один раз; UI спрашивает его через IPC, а не повторяет.
_Avoid_: pickEffectiveAuth (удалённое TS-зеркало)

**Auth materialization (материализация auth)**:
Превращение выбранного конфига в конкретный заголовок: EnvVar — чтение OS-энва;
OAuth2 — `{{var}}`-резолв полей конфига и получение токена через token source.
Асинхронна (в отличие от pick).
_Avoid_: resolve_auth (старое одношаговое имя, объединявшее pick и materialize)

**Token source**:
Шов получения OAuth2-токена по уже разрешённому конфигу. Два адаптера: боевой
провайдер с токен-кэшем и тестовый фейк (oauth-сценарии резолва тестируются без
сети).
_Avoid_: token provider (имя конкретного боевого адаптера, не шва)

### Отправка

**Send (спайн отправки)**:
Единственная точка «запрос/черновик + коллекция + окружение → выполненный вызов +
send report»: резолв-пайплайн → экспансия билтинов → инъекция материализованного
auth-заголовка → активация через contract cache → invoke → инвалидация токен-кэша
на UNAUTHENTICATED. Отмена и
таймаут — не часть Send, а забота вызывающего слоя.
_Avoid_: оркестрация в IPC-команде (командный слой — адаптер над Send, не дом спайна)

### Вызов

**Call options**:
Пер-вызовные опции invoke (timeout, лимит размера сообщения, …), путешествующие
от UI до транспорта одним растущим значением, а не позиционными параметрами.
Ключ отмены (request id) — не call option: у него свой жизненный цикл.
_Avoid_: перечисление опций россыпью в сигнатурах
