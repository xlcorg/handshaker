# Frontend (`src/`) — контекст

React 18 UI. Говорит с ядром только через фасад `src/ipc/client.ts`
(единственный потребитель генерируемого `src/ipc/bindings.ts`).

## Language

**Body completion**:
Чистый ответ на вопрос «что покажет suggest-виджет в этой позиции тела запроса» —
`computeCompletion(fullText, caretOffset, {schema, vars})` в
`src/features/bodyview/completion.ts`. Единственный дом ветвления var-vs-schema,
range-математики и правил вставки; Monaco-регистрация и auto-trigger в BodyView —
только потребители (`source: "vars" | "schema" | null`).
_Avoid_: повторная реализация ветвления в провайдере или обработчиках клавиш.

**Edit buffer (буфер правки)**:
Редактируемый блок Collection Overview (Variables, Links, Auth) держит правку в
локальном буфере: persist на каждое изменение, но обратно из стора буфер
пересеивается только при смене коллекции — эхо persist→reload и внешние изменения
открытую правку не затирают.
_Avoid_: полностью управляемое поле, чьё значение выводится из стора на каждый рендер.

**Empty auth header/prefix**:
Пустой Header name в auth-конфиге — состояние только буфера правки. При persist
пусто → дефолт вида (`authorization` / `x-api-key`); стор пустого header name
не хранит, нормализация — момент сохранения, не момент ввода. Обратная сторона
той же нормализации — **placeholder seeding**: `configToForm` сеет поле Header name
пустым, когда сохранённое значение равно дефолту вида, так что дефолт показывается
серым placeholder'ом, а не как ввод пользователя. Prefix при этом **не**
placeholder-mapped: пустой prefix round-trip'ится как пустой («no prefix» — значимое
состояние), поле prefix у API key всегда видимо и редактируемо.
_Avoid_: показывать дефолтный header как значение поля или прятать сохранённый prefix.

**Bearer — presentation label**:
Bearer не хранится как отдельный вид: `env_var` с header `authorization` + prefix
`Bearer ` рендерится как Bearer, в какой бы вкладке его ни ввели. Одна каноническая
пара → одно каноническое представление; переворот вкладки после re-open принят.
_Avoid_: хранить «выбранную вкладку» — вид выводится из пары header/prefix.

**Смена вида auth — риск затирания (accepted risk)**:
Переключение вида авторизации persist'ится немедленно, и стор держит только
активный вариант конфига; поля неактивного варианта не сохраняются параллельно.
Единственная защита — локальный буфер правки, пересеиваемый только при смене
коллекции. Toast/undo нет — потеря полей прежнего вида при переключении принята
сознательно.
_Avoid_: undo-стек или хранение всех вариантов «на всякий случай» — стор всегда
про один активный вариант.

**«Get token» vs Send — область резолва (accepted risk)**:
Кнопка «Get token» резолвит `{{vars}}` в OAuth2-полях только против активного
окружения; Send те же поля резолвит против env + переменных коллекции. Расхождение
областей принято сознательно — «Get token» сохраняет строгое (env-only) поведение
тест-воркфлоу.
_Avoid_: считать подсветку в OAuth2-полях (env + collection) обещанием того, что
разрешит «Get token».
