**Goal:** Bring the collection sidebar to Postman/shadcn parity — full migration of the tree
onto the canonical shadcn `Sidebar` (+ `ResizablePanelGroup` for width), with backend
persistence of expansion / sort / active request, a configurable gRPC icon, 8px indentation,
guide lines, an icon sort control, refined click behavior, active-row highlight, and
auto-expand-on-drag.

**Architecture:** Backend gains an `expanded` flag on `Collection`/`Folder` (persisted in
collection JSON) + a tiny `ui-state.json` app-settings store; both surfaced via new IPC. The
frontend vendors shadcn `sidebar.tsx` (edited to drop cmd+B/cookie/mobile-Sheet), wraps the
`[sidebar | main]` layout in `ResizablePanelGroup`, and rewrites the tree
(`CollectionTree`/`CollectionNode`/`FolderNode`/`RequestRow`) onto shadcn menu primitives while
keeping the domain model (`useCatalogTree`) and DnD planning (`dnd.ts`) intact. gRPC icon style
is a new localStorage pref.

**Tech Stack:** Rust (handshaker-core `persist`/`tree`, tauri-specta), TypeScript, React 18,
shadcn/ui (Radix + Tailwind v4 + CVA), lucide-react, react-resizable-panels, Vitest + Testing
Library. `@/` path alias = `src/`.

## Decisions locked from spec

1. **Full shadcn migration (level 3)** + `ResizablePanelGroup` for width. cmd+B/cookie/Sheet
   cut from the vendored `sidebar.tsx` (we own the file). Visibility stays `prefs.sidebar` +
   cmd+ctrl+B; no icon-rail.
2. **Persistence split:** expansion + sort + active request → backend; `grpcIcon` → prefs.
3. **`expanded` default `false`**; `#[serde(default)]` keeps existing collection JSON loading.
4. **Indent 8px**; guide lines via `SidebarMenuSub` border; active via `isActive`.
5. **Click:** chevron toggles; collection/folder name opens overview + expands (idempotent);
   request row click opens (except action slot / rename / drag handle).
6. **DnD:** keep `planDrop` logic; add ~700ms auto-expand on hover (persists via same toggle path).
