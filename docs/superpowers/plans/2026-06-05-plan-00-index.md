## File-structure decomposition (locks in boundaries)

### Backend (Rust) — plan-01 / plan-02

- `crates/handshaker-core/src/collections/mod.rs` — `Collection`/`Folder`/`SavedRequest`
  structs: drop `auth_by_env`, add single `auth`, `MetadataRow` metadata, `pinned`,
  `description`, `created_at`, `last_used_at`, `use_count`. New `MetadataRow` struct.
- `crates/handshaker-core/src/collections/resolve.rs` — metadata loop over `Vec<MetadataRow>`
  (skip `enabled == false`); auth chain `request.auth → collection.auth → None` (folders
  carry no auth).
- `crates/handshaker-core/src/collections/file_store.rs` + `in_memory.rs` — unchanged logic;
  tests updated for new fields.
- `src-tauri/src/ipc/collection.rs` — DTOs: `SavedRequestIpc`/`FolderIpc`/`CollectionIpc`
  gain/lose fields; remove `AuthByEnvIpc`; `MetadataRowIpc`. `from_core`/`into_core`.
- `src-tauri/src/commands/collection.rs` — remove `auth_set_for_env`; add
  `collection_set_node_auth` (collection|request) and `collection_bump_usage` (Send hook).
- `src-tauri/src/state.rs` — new persistent contract cache store (plan-02).
- `crates/handshaker-core/src/grpc/file_contract_cache.rs` (NEW, plan-02) — file-backed
  `ContractCache` impl, next to its trait; keyed by resolved address. (Plan-02 moved this
  from the originally-listed `collections/` — the cache is a grpc concern.)
- `src/ipc/bindings.ts` — regenerated.

### Frontend — plan-03..09 (target file map)
