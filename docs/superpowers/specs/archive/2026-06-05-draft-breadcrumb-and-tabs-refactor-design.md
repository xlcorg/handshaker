# Рефакторинг черновика запроса — полный путь в крошке + единый таб-стрип

**Статус:** дизайн утверждён, готов к плану.
**Дата:** 2026-06-05.
**Ветка:** `redesign/workflow-ui-spec-plans`.
**Режим исполнения:** subagent-driven (дефолт).

## 1. Контекст и проблема

Экран черновика запроса (Focus) состоит из шапки-крошки → адрес+method picker → табы
Request/Metadata/Auth → тело и панель ответа. Две конкретные UX-претензии:

1. **Крошка показывает только имя реквеста**, а не полный путь с коллекцией (и папками).
   На скрине вместо `Коллекция › Папка › Create` видно голое `Create`.
2. **Таб-стрипы запроса и ответа стоят на разных вертикальных уровнях и в разном стиле**:
   слева низкие «пилюли», справа более высокий ряд с подчёркиванием.

### Корни в коде

- **Крошка.** [`draftBreadcrumb`](../../../src/features/workflow/draftHeader.ts) *умеет*
  формат `Коллекция › Имя`, но откатывается на голое имя, потому что при открытии
  сохранённого реквеста ([`openSavedRequest`](../../../src/features/catalog/actions.ts))
  `origin` создаётся без `collectionName`. Кроме того, даже `Коллекция › Имя` не
  показывает **папки** — коллекции вложенные (collection → folder → request). Есть
  готовый [`pathToItem`](../../../src/features/catalog/treeNav.ts), отдающий путь
  контейнеров по `requestId`.
- **Табы.** [`RequestTabs`](../../../src/features/workflow/RequestTabs.tsx): контейнер
  `px-2 py-1` (низкий, без фикс. высоты), активный таб — «пилюля» `bg-accent`.
  [`ResponsePanel`](../../../src/features/response/ResponsePanel.tsx): контейнер
  `h-10 px-3.5` + `backdrop-blur`, табы — компонент
  [`UnderlineTabs`](../../../src/components/ui/underline-tabs.tsx) (подчёркивание).
  Отсюда разная высота (→ разный уровень), отступы и активный стиль.

## 2. Решение

### A. Полный путь в хлебной крошке — `Коллекция › Папки › Реквест`

- **`draftHeader.ts`** → `draftBreadcrumb(draft, origin, collections)` остаётся чистой
  функцией, но получает третьим аргументом дерево каталога:
  - **unbound** → `«Новый реквест»` (перевод с английского `New request` —
    рядом уже русское «Сохранено»).
  - **bound** → `pathNamesToItem(collections, origin.requestId)` даёт полный массив
    имён `[collection, …folders, request]` → склейка через ` › `.
  - **fallback** → если id не найден в каталоге (только что сохранён, до `reload`;
    либо удалён) — собираем `origin.collectionName` / `origin.requestName`, как сейчас.
    Поля `DraftOrigin` остаются ровно ради этой подстраховки; save-флоу не меняется.
- **Хелпер в `treeNav.ts`**: `pathNamesToItem(collections, itemId): string[] | null` —
  полный путь именами **включая сам целевой элемент** (для реквеста — это
  `[коллекция, …папки, имя реквеста]`). Симметричен существующему `pathToItem`
  (тот отдаёт id контейнеров до элемента), но возвращает имена и завершает их именем
  самого элемента — благодаря чему крошка остаётся верной даже при переименовании
  самого реквеста. Отдельно тестируется.
- **Источник истины — живой каталог**, а не снапшот в `origin`: крошка остаётся
  верной после переименования/перемещения реквеста при открытом черновике.
  [`FocusView`](../../../src/features/workflow/FocusView.tsx) берёт дерево через
  `useCatalog()` и передаёт в `draftBreadcrumb`.
- **Усечение длинного пути** (в `FocusView`): последний сегмент (имя реквеста) —
  `flex-none`, всегда видим; префикс `Коллекция › Папки ›` — `min-w-0 truncate`,
  клипается при нехватке ширины. Самое важное (имя) не пропадает.

### B. Единый таб-стрип запроса и ответа — стиль «подчёркивание»

- **`RequestTabs.tsx`**: самописные кнопки-«пилюли» заменяются на компонент
  [`UnderlineTabs`](../../../src/components/ui/underline-tabs.tsx); контейнер
  приводится к шаблону панели ответа — `h-10`, `px-3.5`, та же `border-b` и фон.
  Обе строки табов встают на один вертикальный уровень и в одном стиле.
- **`ResponsePanel`** — эталон, **не трогаем**.

## 3. Затрагиваемые файлы

| Файл | Изменение |

## 4. Тесты (TDD, до реализации)

- `treeNav.test.ts` — `pathNamesToItem`: коллекция-корень, вложенные папки,
  несуществующий id (`null`).
- `draftHeader.test.ts` — полный путь из каталога; fallback на `origin`-имена когда
  id вне каталога; unbound-лейбл `«Новый реквест»`.
- `RequestTabs.test.tsx` — рендерит `UnderlineTabs` (роль `tablist`, переключение
  табов, активное подчёркивание вместо `bg-accent`).
- `FocusView.test.tsx` — крошка строится из каталога; усечение префикса при
  сохранённом видимом имени реквеста.

## 5. Вне скоупа (YAGNI)

- Удаление `collectionName` / `requestName` из `DraftOrigin` — нужны как fallback
  до перезагрузки каталога.
- Любые правки бэкенда.
- Изменения `DraftAddressBar`, логики reflection, save-диалога, панели ответа.

## 6. Критерии готовности

- Открытый из сайдбара/overview сохранённый реквест показывает в шапке
  `Коллекция › …папки… › Имя`; имя не пропадает при узкой ширине.
- Только что сохранённый (до `reload`) показывает хотя бы `Коллекция › Имя` (fallback).
- Несохранённый черновик показывает `«Новый реквест»`.
- Табы Request/Metadata/Auth и Body/Trailers/Headers визуально на одном уровне,
  оба в стиле подчёркивания.
- Все новые и существующие тесты зелёные.
