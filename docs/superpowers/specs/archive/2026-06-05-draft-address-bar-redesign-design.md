# Редизайн раскладки окна черновика (DraftAddressBar)

**Статус:** дизайн утверждён, готов к плану реализации
**Дата:** 2026-06-05
**Тип:** визуальный/раскладочный рефакторинг (без изменения бэкенда и модели данных)
**Затрагивает:** `src/features/workflow/{FocusView,CallPanel,DraftAddressBar}.tsx`,
`src/features/shell/MethodPicker.tsx`

## Контекст и мотивация

Окно черновика (`FocusView` → `CallPanel` с `editable`) — это request-first
поверхность создания вызова. Текущая раскладка адрес-бара тесная и неудачно
распределяет ширину:

- host-инпут зафиксирован узким (`w-56`), а `MethodPicker` зажат в середине;
  `flex-1`-спейсер отталкивает Send вправо, оставляя пустоту;
- когда контракт ещё не отражён, вместо пикера показывается plain-текст
  («Enter a host to load methods» / «Reflecting…» / текст ошибки);
- кнопка обновления контракта (`RefreshCw`) торчит отдельной иконкой в баре;
- переключения TLS в черновике нет вовсе — `step.tls` правится только из других
  мест; в самом окне флаг невидим;
- состояние сохранения живёт в отдельной тонкой полосе (28px) над `CallPanel`
  с текстом «Сохранено» / кнопкой «Сохранить».

Цель — привести верх окна черновика к раскладке gRPC-запроса Postman: одна
строка адреса, где host компактен слева, селектор метода растягивается на всё
свободное место, Send справа; обновление контракта и статус рефлексии убраны
внутрь дропдауна метода; TLS — тихий замок внутри поля host; сохранение —
кнопка-иконка в хедере, отдельная полоса статуса убрана.

Референс: gRPC-клиент Postman — верхняя строка `Server URL` → дропдаун
`Select a method` (группировка по сервису, тип метода иконкой) → `Invoke`;
рефлексия подгружается автоматически, её статус и перезагрузка — внутри
дропдауна («Using server reflection ⟳»).

## Объём (scope)

**В объёме:** раскладка и визуал верха окна черновика — хедер, строка адреса,
дропдаун метода. **Вне объёма:** нижний сплит Request | Response (не меняется),
бэкенд, модель `Step`, логика автосейва/Save-диалога (только перенос триггера в
хедер), `skip_verify` (остаётся `false`).

## Целевая раскладка

Сверху вниз окно черновика состоит из трёх горизонтальных полос (хедер → строка
адреса → нижний сплит без изменений):

### 1. Хедер (заменяет полосу статуса автосейва)

Высота ~36px, `border-b`. Заменяет нынешнюю 28px-полосу в `FocusView`.

- **Слева — брэдкрамб:** несохранённый черновик → приглушённое `New request`;
  привязанный (есть `useDraftOrigin()`) → `<Коллекция> › <Имя>`.
- **Справа — кнопка Save:** иконка (`Save` из lucide) + подпись «Save».
  Рядом индикатор «грязноты»: маленькая жёлтая точка = есть несохранённые
  изменения; точки нет = сохранено. Клик по несохранённому черновику открывает
  Save-диалог (`onRequestSave`), как сейчас кнопка «Сохранить». Поведение
  автосейва привязанного черновика сохраняется — меняется только подача.

### 2. Строка адреса (`DraftAddressBar`)

Одна строка ~46px, `border-b`, элементы слева направо:

1. **Поле host** — компактное, фиксированной ширины (ориентир ~190–230px),
   моно-шрифт. **Внутри поля слева** — тихая иконка-замок TLS:
   - `Lock` (закрыт) = TLS включён; `Unlock` (открыт) = plaintext;
   - вид приглушённый (`text-muted-foreground`), **без рамки и цветового
     акцента** — иконка не должна привлекать внимание;
   - клик переключает `step.tls` (`onPatch({ tls })`); тултип как в
     `ConnectionBar` («TLS enabled — click to switch to plaintext» /
     «Plaintext — click to enable TLS»).
2. **Селектор метода** — `MethodPicker`, **растягивается** на всё свободное место
   (`flex-1`). Виден всегда, в т.ч. без контракта:
   - метод не выбран → плейсхолдер `Select a method`;
   - метод выбран → `↑↓ <Service> / <Method>` (как сейчас: kind-иконка +
     короткое имя сервиса + метод).
3. **Кнопка Send** — справа, акцентная (primary). Disabled, пока метод не выбран
   или идёт отправка; во время отправки рядом — Cancel (как сейчас).

Прежний отдельный `RefreshCw`-кнопка из бара **удаляется** (переезжает в дропдаун,
см. ниже). Прежний `flex-1`-спейсер удаляется — его роль играет растягивающийся
`MethodPicker`.

### 3. Дропдаун метода (`MethodPicker`) — доводка

`MethodPicker` уже Postman-образный (поиск `Find service.method…`, группировка по
сервису, `KindDot`/`KindBadge` по типу метода). Доводки для черновика:

- **Работает без каталога:** принимает `catalog: ServiceCatalogIpc | null`. При
  `null` триггер показывает плейсхолдер `Select a method`; в раскрытом виде —
  только футер рефлексии (см. ниже), список методов пуст.
- **Футер рефлексии** (опционально, только в режиме черновика): нижняя строка
  дропдауна со статусом и перезагрузкой контракта:
  - идёт рефлексия → «Reflecting…» (со спиннером);
  - ошибка → текст ошибки (`reflectError`);
  - есть каталог → «Using server reflection» + кнопка `⟳` (вызывает
    `reflection.refresh()` — это перенесённая `RefreshCw`-кнопка).

  Футер рендерится, только когда переданы reflection-пропсы (`reflecting`,
  `reflectError`, `onRefresh`), чтобы не задеть другие места использования
  `MethodPicker` (shell).

## Изменения по компонентам

- **`FocusView.tsx`** — заменить 28px-полосу статуса на хедер (§1): брэдкрамб
  слева, Save-кнопка-иконка + dirty-точка справа.
- **`DraftAddressBar.tsx`** — перекомпоновать строку (§2): замок TLS внутри
  host-поля; host фикс-ширины; `MethodPicker` на `flex-1` и всегда видим; убрать
  отдельную `RefreshCw`-кнопку и текстовый статус-плейсхолдер; пробросить
  reflection-пропсы в `MethodPicker`; добавить `onTls`-проп.
- **`CallPanel.tsx`** — прокинуть `onTls={(tls) => onPatch({ tls })}` в
  `DraftAddressBar`; передать `reflection.{loading,error,refresh}` дальше в
  `MethodPicker` через `DraftAddressBar`.
- **`MethodPicker.tsx`** — `catalog` сделать nullable; плейсхолдер-триггер;
  опциональный футер рефлексии за новыми пропсами; существующие вызовы (shell)
  остаются рабочими (футер не рендерится без пропсов).

## Поведение и крайние случаи

- **Нет адреса:** host пуст → `MethodPicker` показывает `Select a method`,
  дропдаун — пустой список без футера-статуса (рефлексия не запускается; как
  сейчас `useDraftReflection` no-op на пустом адресе). Send disabled.
- **Адрес есть, рефлексия идёт:** триггер `Select a method`, в дропдауне футер
  «Reflecting…».
- **Ошибка рефлексии:** триггер `Select a method`, футер показывает ошибку;
  пользователь может нажать `⟳` для повтора (через `grpcRefreshContract`).
- **TLS-переключение:** меняет `step.tls`; `useDraftReflection` дебаунс-перезапрос
  по `(address, tls)` уже реагирует на смену `tls` — контракт перечитается
  автоматически.
- **Метод выбран, потом адрес сменили:** существующее поведение сохраняется
  (контракт перечитывается; выбранный метод остаётся в `step` до новой выборки).

## Тестирование

Юнит-тесты (Vitest + RTL), в духе существующих `DraftAddressBar.test.tsx`,
`CallPanel.editable.test.tsx`, `FocusView.test.tsx`:

- замок TLS: рендер `Lock`/`Unlock` по `step.tls`; клик зовёт `onTls`/`onPatch`
  с инвертированным значением; есть `aria-label`/тултип;
- `MethodPicker` без каталога: триггер показывает `Select a method`; Send disabled;
- футер рефлексии: «Reflecting…» при loading; текст ошибки при error; «Using
  server reflection» + `⟳`, и клик зовёт `onRefresh`;
- футер **не** рендерится, когда reflection-пропсы не переданы (регресс shell);
- хедер `FocusView`: брэдкрамб `New request` без origin и `Коллекция › Имя` с
  origin; Save-кнопка с dirty-точкой; клик зовёт `onRequestSave`;
- строка адреса: `MethodPicker` растянут (`flex-1`); отдельной `RefreshCw`-кнопки
  в баре больше нет.
