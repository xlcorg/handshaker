# Редизайн Save-диалога: Postman-style пикер + рекомендация по структуре сервера

**Дата:** 2026-06-05
**Статус:** дизайн (ожидает ревью пользователя → плана реализации)
**Тип:** UX/layout-редизайн `SaveRequestDialog` (фронтенд) + новая чистая функция деривации в `grouping.ts`
**Ветка:** `redesign/workflow-ui-spec-plans`

> Предшественник — рефакторинг коллекции сервисов
> (`2026-06-05-service-collection-sidebar-refactor-design.md`, plan-06/plan-10).
> Save-flow интеграция там **завершена**; этот спек **переписывает только UI/UX**
> диалога и добавляет авто-рекомендацию места сохранения. Бэкенд и
> `useCatalogTree`/`saveNewRequest` — без изменений.

---

## 1. Контекст и проблема

Текущий `SaveRequestDialog` (`src/features/catalog/SaveRequestDialog.tsx`)
функционален, но раскладка неудобна:

1. **Два отдельных `<select>`** (коллекция + папка) вместо единого навигируемого
   выбора места в дереве.
2. **Нельзя создать вложенную папку** прямо в диалоге — только коллекцию верхнего
   уровня.
3. **«Suggested path»** — только текст-подсказка (`Host › Service`), по ней нельзя
   кликнуть/применить.
4. **Маленький фиксированный размер** — в большой коллекции искать папку неудобно.

**Цель:** Postman-подобный пикер места сохранения (видимое навигируемое дерево
коллекций/папок + поиск + инлайн-создание) **плюс** умная авто-рекомендация:
структура сохранения зеркалит структуру сервера — gRPC `NotesApiService/Create`
рекомендуется как папка `NotesApi`, имя запроса `Create`.

## 2. Решения (одобрено пользователем)

- **Подача рекомендации — вариант A (чип-подсказка), не авто-наведение.** Дерево по
  умолчанию НЕ навязывает выбор; рекомендация живёт в отдельном чипе с явной кнопкой
  «Добавить».
- **Деривация папки:** последний dot-сегмент `service` минус трейлинг-суффикс
  `Service`. `notes.v1.NotesApiService` → `NotesApi`. Если суффикса `Service` нет —
  короткое имя как есть.
- **Имя запроса:** короткое имя метода (`Create`).
- **Хост в путь не входит** — только сервис→папка.
- **Дефолтная коллекция** для рекомендации (когда выбор пуст) = **первая в списке**.
- **Без пометки `(new)`** — создаваемая папка показывается как обычный сегмент пути;
  создаётся молча при Save.
- **Единая контекстная кнопка «＋ New»** вместо раздельных new-collection / new-folder.
- **Нижний предпросмотр «Save to» убран** — цель видна по выделению в дереве.
- **Размер диалога:** высота `80vh`, ширина ≈640px; скроллится только панель дерева.

## 3. Раскладка (UX)

Диалог = вертикальная флекс-колонка фиксированной высоты `80vh`. Сверху вниз:

1. **Request name** — `<input>`, префилл = рекомендованное имя запроса (имя метода).
2. **Чип-рекомендация** (показывается только когда у черновика известны `service` +
   `method`):
   `✨ Рекомендуем сохранить как: <коллекция> / NotesApi / Create` + кнопка
   **«Добавить»**.
3. **«Already saved in»** (опц.) — ненавязчивая подсказка под чипом, если этот вызов
   уже где-то сохранён (`findSavedLocations`).
4. **Поиск** — `<input>`, фильтрует дерево по имени коллекции/папки.
5. **Дерево** (`flex:1; overflow:auto; min-height:0`) — навигируемый список
   коллекций → папок; одиночный выбор целевого узла; раскрытие/сворачивание.
   Read-only по содержимому (без DnD/rename/delete/pin — это не сайдбар).
6. **«＋ New»** — контекстная кнопка (слева внизу).
7. **Footer** — Cancel / Save.

Закреплены: имя, чип, «already saved in», поиск, «＋ New», footer. Скроллится только
панель дерева.

### 3.1 Чип-рекомендация

- Полный путь = `<текущая выбранная коллекция> / <derivedFolder> / <requestName>`.
  Коллекция берётся из текущего выбора в дереве; если ничего не выбрано — первая в
  списке.
- Кнопка **«Добавить»**: создаёт (виртуально, до Save) и выбирает папку
  `derivedFolder` в текущей коллекции; путь применяется к выбору в дереве.
- Если папка `derivedFolder` уже существует в коллекции — «Добавить» просто наводит
  в неё, без дублей.

### 3.2 Контекстная «＋ New»

Лейбл и действие зависят от текущего выбранного узла:

- выбран **корень** (ни одна коллекция) → `＋ New collection`;
- выбрана **коллекция** «X» → `＋ New folder in "X"`;
- выбрана **папка** «Y» → `＋ New folder in "Y"` (вложенная).

Папки на верхнем уровне невозможны — там только коллекции.

### 3.3 Режим `originBound` («Update request»)

Без изменений: когда реквест уже привязан к коллекции, диалог показывает только поле
Name под заголовком «Update request». Пикер/чип/поиск не рендерятся.

## 4. Архитектура / компоненты

### 4.1 `suggestSaveTarget(service, method)` — новая чистая функция

Рядом с `suggestSavePath` в `src/features/catalog/grouping.ts`:

```ts
export interface SaveTarget {
  folderName: string;   // напр. "NotesApi"
  requestName: string;  // напр. "Create"
}
export function suggestSaveTarget(service: string, method: string): SaveTarget;
```

- `folderName` = `serviceShortName(service)` минус трейлинг `Service`
  (регистрозависимо, целиком слово-суффикс; `Service` сам по себе → не пустим в
  пустую строку, оставляем `Service`).
- `requestName` = `method` (как есть; уже короткое имя метода).

Существующий `suggestSavePath` сохраняем, пока его использует кто-то ещё; если после
редизайна он более не нужен — удаляем (проверить ссылки на этапе плана).

### 4.2 `CollectionPicker` — новый лёгкий компонент

Презентационный пикер места сохранения. Не переиспользуем `CollectionTree`
(перегружен DnD/rename/delete/pin/keyboard под сайдбар).

Пропсы (ориентировочно):

```ts
interface CollectionPickerProps {
  collections: CollectionIpc[];            // из metas + ленивая подгрузка
  query: string;                           // строка поиска
  value: PickTarget | null;                // { collectionId, parentId }
  onChange: (t: PickTarget) => void;
  pendingFolders: PendingFolder[];          // виртуальные (ещё не сохранённые) папки
}
```

- Переиспользует чистые хелперы из `treeNav` (`flattenVisible`, `pathToItem`,
  `allContainerIds`) для раскрытия/навигации. Локальный стейт раскрытых узлов.
- Рендерит виртуальные `pendingFolders` как обычные узлы (без пометки).
- При активном `query` — раскрывает всё и фильтрует по имени.

### 4.3 `SaveRequestDialog` — переписывается

- Композирует `CollectionPicker` + чип + поиск + контекстную «＋ New».
- Пропсы в основном прежние (`metas`, `loadCollection`, `onSave`,
  `onCreateCollection`, `originBound`, `existingLocations`).
- **Добавляются** `draftService` / `draftMethod` (для `suggestSaveTarget`).
- **Убираются** оба `<select>` и текст-подсказка `suggestedPath`.
- Размер: `DialogContent` высотой `80vh`, ширина ≈640px, флекс-колонка.

## 5. Данные / поток

1. **Открытие** → `suggestSaveTarget(draftService, draftMethod)` даёт `requestName`
   (префилл Name) и `folderName` (для чипа). Дерево из `metas` + ленивый
   `loadCollection`. Стартовый выбор = первая коллекция (без папки).
2. **Чип** строит полный путь от текущего выбора. «Добавить» → добавляет
   `pendingFolder` `{collectionId, name}` и выбирает её.
3. **«＋ New»** → инлайн-ввод имени; коллекция создаётся через `onCreateCollection`,
   папка — как `pendingFolder` под текущим узлом.
4. **Save** →
   - материализуем `pendingFolders` по порядку (`addFolder` через `useCatalogTree`),
     получая реальные id;
   - вызываем `onSave({ collectionId, parentId, name })` — существующий
     `saveNewRequest` без изменений.

## 6. Граничные случаи

- **Нет коллекций вообще** → дерево пустое; «＋ New» = New collection; чип предлагает
  `<новая коллекция> / NotesApi / Create` (коллекция материализуется при Save).
- **Черновик без метода** (`method` пуст) → чип скрыт; Name = текущее имя/`Untitled`.
- **`folderName` совпадает с существующей папкой** → «Добавить» наводит в неё, дублей
  не создаём.
- **`service` без суффикса `Service`** → папка = короткое имя как есть.
- **Дубль-детект** (`findSavedLocations`) → «Already saved in» как подсказка, не
  блокирует сохранение.

## 7. Тестирование

**Юнит (`grouping.test.ts`):**
- `suggestSaveTarget`: суффикс `Service` отбрасывается; без суффикса — как есть;
  пакетное имя берёт последний сегмент; пустой `method`.

**Компонент (RTL, `SaveRequestDialog.test.tsx`):**
- Name префиллится именем метода.
- Чип строит полный путь от выбранной коллекции; «Добавить» меняет путь/выбор.
- Контекстная «＋ New»: лейбл зависит от выбора (root → collection, иначе folder).
- Поиск фильтрует дерево.
- Save материализует `pendingFolders` и зовёт `onSave` с верным `parentId`.
- `originBound`: только поле Name, пикер не рендерится.

## 8. Вне scope (YAGNI)

- Поле «Add description» (есть в Postman) — не добавляем.
- Хост как уровень папки — отклонено.
- Авто-наведение/авто-создание без явного «Добавить» — отклонено (вариант B).
- Изменения бэкенда, `useCatalogTree`, `saveNewRequest`, `autosaveDraft`.
